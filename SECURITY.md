# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please follow these steps:

1. **Do not create a public issue** — this avoids exposing the vulnerability to the public.
2. **Send an email** to the repository maintainer at: `contact@aymanemehdi.com`  
3. Include the following details:
   - A clear description of the vulnerability
   - Steps to reproduce
   - Any relevant screenshots or logs
4. The maintainer will acknowledge your report within 48 hours and provide updates on the fix.

---

## Security Best Practices

- Keep dependencies up to date
- Use strong passwords and environment variable secrets
- Validate and sanitize all user input
- Use HTTPS for all communications
- Regularly review and audit code for vulnerabilities
